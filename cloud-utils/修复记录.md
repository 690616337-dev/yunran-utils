# 修复记录

## 修复的问题

### 1. 身份证查验 - 验证失败显示 `[object Object]`
**问题原因**: 错误处理时直接将对象转为字符串
**修复文件**: `frontend/src/pages/IdCardPage.jsx`
**修复内容**: 
- 使用 `error.userMessage` 优先显示友好的错误信息
- 添加了默认错误信息 '未知错误'

### 2. 身份证生成器 - 日期选择器改进
**问题原因**: 用户希望可以手动输入日期
**修复文件**: `frontend/src/pages/IdCardPage.jsx`
**修复内容**:
- 添加了 `format="YYYY-MM-DD"` 属性
- 添加了 `allowClear` 属性允许清空
- 更新了 placeholder 提示文字

### 3. 证件照抠图 - 导入后无预览图
**问题原因**: 上传图片后没有生成预览URL
**修复文件**: `frontend/src/pages/PhotoPage.jsx`
**修复内容**:
- 在 `handleUpload` 中创建 `URL.createObjectURL` 预览链接
- 在卡片封面显示原图预览
- 添加了 `removeFile` 和 `clearAll` 中清理预览URL的逻辑，避免内存泄漏
- 改进了错误处理，显示更清晰的错误信息

### 4. 音频转换 - 400 错误
**问题原因**: 比特率参数处理可能有问题
**修复文件**: 
- `backend/main.py` - 改进了比特率参数处理逻辑
- `frontend/src/pages/AudioPage.jsx` - 改进了错误信息显示
**修复内容**:
- 后端：确保比特率参数正确转换为字符串并添加 'k' 后缀
- 前端：使用 `error.userMessage` 显示更友好的错误信息

## 测试建议

1. **身份证查验**: 输入无效身份证，检查错误提示是否正常显示
2. **身份证生成器**: 测试日期选择器的手动输入和选择功能
3. **证件照抠图**: 上传图片后检查是否显示预览图，处理完成后检查结果显示
4. **音频转换**: 测试不同格式的音频转换，检查错误提示是否清晰
