# 精雀赛事数据获取问题研究报告

## 一、API接口分析

### 1.1 GraphQL端点分析

**端点地址**: `https://service.danceapi.com/graphql`

**已确认的API结构**:
```
Query类型字段:
- event(id): 查询单个赛事（需要登录）
- delegations(options): 查询代表团列表（需要登录）
- securityToken: 获取安全令牌
- getMatchFilesSignature: 获取文件签名

Match类型字段（共80+字段）:
- id, name, status
- matchTimeStart, matchTimeEnd
- locationName, locationCity
- organizerName, organizerId
- banner, logo
- enrollTimeStart, enrollTimeEnd
- ...（其他管理字段）
```

### 1.2 接口测试结果

| 测试接口 | 结果 | 说明 |
|---------|------|------|
| `__schema` 内省查询 | ✅ 成功 | 可获取完整API结构 |
| `event(id)` | ❌ 失败 | 返回"未登录"错误(401) |
| `matches` 列表查询 | ❌ 失败 | 字段不存在 |
| `events` 列表查询 | ❌ 失败 | 字段不存在 |
| REST API `/api/matches` | ❌ 失败 | 404 Not Found |

### 1.3 结论

**精雀赛事没有提供公开访问赛事列表的API接口**。所有赛事数据查询都需要用户登录认证后才能访问。
